【API 接口清单】

1. GET /api/videos
   └─ 获取所有视频文件列表

2. POST /api/cast
   └─ 投屏视频（调用go2tv.py）

3. POST /api/delete
   └─ 删除视频文件

4. GET /api/devices
   └─ 获取DLNA设备列表（可选）


【快速开始】

步骤1：创建虚拟环境
  $ cd /Users/one/Documents/项目/测试1
  $ python3 -m venv venv
  $ source venv/bin/activate

步骤2：安装依赖
  $ pip install -r requirements.txt

步骤3：准备视频文件
  $ mkdir -p movie
  $ cp /path/to/video.mp4 movie/

步骤4：启动Web服务
  方式A（推荐）：
    $ bash start_web.sh
  
  方式B（手动）：
    $ source venv/bin/activate
    $ python index.py

步骤5：访问页面
  打开浏览器：http://localhost:5000

【功能演示】

页面加载流程：
  1. 用户访问 http://localhost:5000
  2. 浏览器加载 index.html
  3. JavaScript 调用 /api/videos
  4. 后端扫描 movie 目录
  5. 返回视频列表 JSON
  6. 前端动态渲染表格
  7. 用户可点击操作按钮

投屏流程：
  1. 用户点击[投屏播放]按钮
  2. 前端发送 POST /api/cast 请求
  3. 后端执行 go2tv.py 脚本
  4. 脚本将视频投屏到设备
  5. 返回成功/失败信息
  6. 前端显示操作结果

删除流程：
  1. 用户点击[删除]按钮
  2. 浏览器显示确认对话框
  3. 用户确认后发送 POST /api/delete 请求
  4. 后端删除 movie 目录中的文件
  5. 返回删除结果
  6. 前端自动刷新视频列表

【常见问题】

Q1: 如何添加更多视频？
A: 将 MP4 等视频文件复制到 movie 目录，刷新页面即可。

Q2: 如何修改监听端口？
A: 编辑 index.py 最后一行，修改 port 参数。

Q3: 如何从其他电脑访问？
A: 启动后访问 http://<服务器IP>:5000

Q4: 如何停止服务？
A: 按 Ctrl+C

Q5: 投屏失败怎么办？
A: 检查 go2tv.py 是否存在，参数是否正确。
